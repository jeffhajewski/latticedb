embedded single file graph + vector + bm25

Embedded graph database for local apps that need graph traversal, vector search, and full-text search.

LatticeDB replaces a stack of separate local systems with one embedded database file: property graph storage, HNSW vector search, BM25 full-text search, and durable streams.

Install

Start with the interface you want to use. The shortest path is the CLI. Python and TypeScript packages are expected to bundle the native library on supported platforms. Go currently uses an installed-prefix or repo-local cgo workflow.

packages Choose a binding.
  • CLI for importing data, running queries, and inspecting files.
  • Python for local applications, notebooks, and scripts.
  • TypeScript for Node.js services and tooling.
  • Go for cgo-backed native workflows.
CLI 
curl -fsSL https://raw.githubusercontent.com/jeffhajewski/latticedb/main/dist/install.sh | bash

Fastest way to create a database file and run the quickstart.

Python 
pip install latticedb

For local applications, notebooks, and scripts.

TypeScript / Node.js 
npm install @hajewski/latticedb

For Node services and tooling.

Go 
zig build install --prefix /tmp/lattice-install
export PKG_CONFIG_PATH=/tmp/lattice-install/lib/pkgconfig
# macOS
export DYLD_LIBRARY_PATH=/tmp/lattice-install/lib
# Linux
export LD_LIBRARY_PATH=/tmp/lattice-install/lib
cd bindings/go && go test ./...

For repo-local development, use go test -tags repolocal ./....

Five-minute path.

Build or install the CLI, create a file, import the sample graph, and run a query.

step 01
Create a database.

Use the installed CLI or the local binary from zig build.

$LATTICE create /tmp/lattice-quickstart.lattice
step 02
Import the sample graph.

Load the repository’s author/document graph.

$LATTICE import /tmp/lattice-quickstart.lattice \
  --file=examples/cli/people_graph.json
step 03
Inspect counts.

Confirm the file exists and the import worked.

$LATTICE count /tmp/lattice-quickstart.lattice
step 04
Run a query file.

Execute the quickstart Cypher query, then run $LATTICE check.

$LATTICE exec /tmp/lattice-quickstart.lattice \
  --file=examples/cli/authored_docs.cypher

Query model

LatticeDB uses one query layer for graph traversal, vector similarity, and BM25 full-text search. This is the simplest way to show what that means.

example One query across graph, vector, and text.
  • Match chunks, documents, and authors as graph structure.
  • Filter by vector distance on the chunk embedding.
  • Filter by BM25 full-text match on document content.
MATCH (chunk:Chunk)-[:PART_OF]->(doc:Document)-[:AUTHORED_BY]->(author:Person)
WHERE chunk.embedding <=> $query_vector < 0.3
  AND doc.content @@ "neural networks"
RETURN doc.title, chunk.text, author.name
ORDER BY chunk.embedding <=> $query_vector
LIMIT 10

Facts

Embedded, single file, single writer, WAL-backed durability.

operating model Core facts.
  • Embedded property-graph database.
  • Single-file storage model.
  • Single-writer model for one owning process.
  • WAL-backed durability for graph writes and streams.
  • Bindings for Python, TypeScript, and Go.
